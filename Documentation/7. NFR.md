## 7. Non-Functional Requirements

### Performance &amp; Scalability

<table>
  <tr>
      <th>Requirement</th>
      <th>Target</th>
      <th>Notes</th>
  </tr>
  <tr>
      <td style="font-weight:bold">API Response Time</td>
      <td>&lt;200ms (p95)</td>
      <td>State transitions, reads</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Throughput</td>

  </tr>
  <tr>
      <td style="font-weight:bold">Audit Log Latency</td>

  </tr>
  <tr>

### Security &amp; Compliance

<table>
  <tr>
      <th>Requirement</th>
      <th>Implementation</th>
  </tr>
  <tr>
      <td style="font-weight:bold">Role-Based Access Control (RBAC)</td>
      <td>Only approvers can approve tickets; only admins can delete rules</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Audit Trail Immutability</td>
      <td>Activity log uses append-only pattern; no updates after creation</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Encryption at Rest</td>
      <td>AES-256 for sensitive fields (comments, requester data)</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Encryption in Transit</td>
      <td>TLS 1.3 for all API calls</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Data Retention Polic</td>
      <td>Soft-deleted tickets purged after 7 years (configurable)</td>
  </tr>
</table>

<br><br>
