## 5. Entity-Relationship Diagram (Mermaid)

```mermaid

erDiagram

    USERS ||--o{ TICKETS : "raises"
    USERS ||--o{ TICKETS : "approves"
    TICKETS ||--o{ ACTIVITY_LOG : "generates"
    TICKETS ||--o{ COMMENTS : "contains"
    USERS ||--o{ COMMENTS : "writes"
    WORKFLOW_RULES ||--o{ TICKETS : "defines"

    USERS {
        int user_id PK "Primary Key"
        string email UK "Unique email"
        string full_name
        string role "Enum: Requester, Approver, Admin"
        string department "e.g., Finance, HR, Ops"
        boolean is_active "Soft-delete flag"
        timestamp created_at
        timestamp updated_at
    }

    TICKETS {
        int ticket_id PK "Primary Key"
        string ticket_number UK "e.g., MEMO-2024-001"
        int requester_id FK "Foreign Key → Users"
        int current_approver_id FK "Foreign Key → Users"
        string title
        text description
        string status "Enum: Raised, InReview, Approved, Rejected, Completed, Archived"
        string urgency_tier "High, Medium, Low"
        timestamp created_at
        timestamp updated_at
        timestamp due_date "SLA deadline"
        timestamp completed_at
        boolean is_archived "Soft-delete flag"
        int workflow_rule_id FK "Foreign Key → WorkflowRules"
    }

    ACTIVITY_LOG {
        int log_id PK "Primary Key"
        int ticket_id FK "Foreign Key → Tickets"
        int actor_id FK "Foreign Key → Users"
        string action "Enum: CREATED, TRANSITIONED, APPROVED, REJECTED, COMMENTED"
        string old_state "Previous state"
        string new_state "New state"
        text notes "Approver comment"
        timestamp action_timestamp "When it happened"
    }

    COMMENTS {
        int comment_id PK "Primary Key"
        int ticket_id FK "Foreign Key → Tickets"
        int author_id FK "Foreign Key → Users"
        text content
        timestamp created_at
    }

    WORKFLOW_RULES {
        int rule_id PK "Primary Key"
        string workflow_name "e.g., LeaveApproval, MemoReview"
        string description
        text state_machine_config "JSON defining states & transitions"
        string approval_chain "JSON: sequential or parallel"
        int max_days_pending "SLA threshold"
        boolean is_active
        timestamp created_at
    }

```



<br>

**Why This Design?**

* **Immutability:** ACTIVITY_LOG is append-only; never updated or deleted

* **Auditability:** Every action tied to an actor and timestamp

* **Scalability:** Tickets and Activity Log can be sharded by ticket_id or created_at

* **Compliance:** Soft-delete flags (is_archived, is_active) preserve history

* **Extensibility:** WORKFLOW_RULES allows admins to define custom workflows without code changes
