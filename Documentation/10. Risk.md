## 11. Risks &amp; Mitigation

<table>
  <tr>
      <th>Risk</th>
      <th>Impact</th>
      <th>Mitigation</th>
  </tr>
  <tr>
      <td style="font-weight:bold">State Machine Logic Bugs</td>
      <td>Tickets stuck in invalid state</td>
      <td>Comprehensive unit + integration tests; state validation before transitions</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Audit Log Growth</td>
      <td>Storage/query performance degradation</td>
      <td>Partitioning by date; archival strategy after 7 years</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Approver Overload</td>
      <td>Priority algorithm not tuned; low-urgency items deprioritized forever</td>
      <td>Quarterly reviews of weights (0.6, 0.4); watchdog alerts if &gt;30 days pending</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Integration Complexity</td>
      <td>Third-party systems fail to call API correctly</td>
      <td>Comprehensive API documentation; SDK examples (Python, JavaScript, Java)</td>
  </tr>
</table>

**Glossary**

<table>
  <tr>
      <th>Term</th>
      <th>Definition</th>
  </tr>
  <tr>
      <td style="font-weight:bold">Ticket</td>
      <td>Atomic unit of work (e.g., a leave request, memo, purchase order)</td>
  </tr>
  <tr>
      <td style="font-weight:bold">State</td>
      <td>Current lifecycle stage of a ticket (Raised, In Review, Approved, etc.)</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Transition</td>
      <td>Movement from one state to another, triggered by an action</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Approver</td>
      <td>User with permission to approve or reject a ticket</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Activity Log</td>
      <td>Immutable, append-only log of all ticket events</td>
  </tr>
  <tr>
      <td style="font-weight:bold">SLA</td>
      <td>Service Level Agreement; time threshold for approvals</td>
  </tr>
  <tr>
      <td style="font-weight:bold">Soft Delete</td>
      <td>Mark as deleted without removing from database (preserves audit trail)</td>
  </tr>
  <tr>
      <td style="font-weight:bold">RBAC</td>
      <td>Role-Based Access Control; permission model tied to user roles</td>
  </tr>
</table>
