## 4. State Transition Diagram (Mermaid)

```mermaid
stateDiagram-v2

    [*] --> Raised

    Raised --> InReview: Auto-assign to Approver 1

    InReview --> ApprovedStep1: Approver 1 accepts
    InReview --> RejectedReview: Approver 1 rejects

    ApprovedStep1 --> InReview2: Escalate to Approver 2

    InReview2 --> Approved: Approver 2 accepts
    InReview2 --> RejectedReview: Approver 2 rejects

    RejectedReview --> Draft: Notify requester; reset to draft

    Draft --> InReview: Requester resubmits

    Approved --> Completed: Final step done; all stakeholders notified
    Completed --> [*]

    InReview --> Escalated: SLA breach (>2 days pending)
    Escalated --> InReview: Admin reassigns to manager

    RejectedReview --> Cancelled: Requester gives up (soft-delete)
    Cancelled --> [*]

    note right of InReview
        Max 5 days pending
        Escalate if breached
    end note

    note right of RejectedReview
        Requester has 14 days
        to resubmit
    end note
```
